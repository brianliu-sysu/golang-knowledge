# **Context**
context 是golang用于控制goroutine的生命周期和传递元数据的标准手段

## **核心哲学和设计理念**
* 显示传递，使得调用链清晰
* 倒置的树，级连取消
* 只传递控制信息，不要传递业务信息
* 并发安全与不可变

## **最佳实践**
* 位置：确保放在函数参数列表的第一个位置，命名为：ctx
* 存储：不要作为成员，作为函数的第一个参数
* 生命周期：创建了context，记得调用cancel取消
* 初始化：不要传递nil，顶层使用Backgroud,不知道要做什么用Todo
* 传值：使用私有成员定义新的类型，对外提供set和get方法（防止key冲突，类似：type UserKey int）
* 内容：只传元数据，不要传业务数据
